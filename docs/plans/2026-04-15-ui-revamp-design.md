# UI Revamp Design — 2026-04-15

## Goal

Replace the current popup UI with a polished, branded interface while preserving every existing behavior: URL resolution, hop chain display, LinkedIn interstitial handling, history persistence, domain health checks, and the light/dark theme toggle.

## Non-goals

- No new features.
- No new permissions, hosts, or dependencies.
- No bundler or build step.
- No changes to `src/background.js`, `src/resolver.js`, `src/domainHealth.js`, or `src/historyStore.js` behavior. Only `popup/` is rewritten, plus a small `summarizeRisk` helper added to `src/domainHealth.js`.

## Visual system

- **Direction:** playful/branded — rounded cards, soft shadows, restrained accent gradient, a bit of personality.
- **Dimensions:** 400px wide, min-height 480px, grows to ~600px before result list scrolls.
- **Type:** system font stack. Two sizes carry the design — 13px body, 20px destination hostname. Tabular numerals where alignment matters.
- **Tokens** (CSS custom properties; dark overrides under `[data-theme="dark"]`):

| `--bg` | `#fafafa` | `#0b0b12` |

The accent gradient appears in only three places: the toolbar wordmark, the primary "Resolve" button, and a 2px top border on the destination hero card.

## Shell layout

Single column, no left rail. Vertical stack:

1. Toolbar — wordmark (gradient text) + history toggle button + theme toggle button.
2. History drawer — collapsed by default, slides down from below the toolbar.
3. Resolve form — label, input, primary button.
4. Status line — `aria-live="polite"`.
5. Result region — destination hero card, then "Check Domain Health" button below.

## Hero result card

Three zones, top to bottom:

**Zone 1 — Verdict header.** 2px gradient strip across the top of the card. 28px neutral globe SVG icon (no third-party favicon fetches — preserves the extension's "don't talk to the destination" privacy stance). If Domain Health has been run, the icon tints with the risk color. "Host changed" amber chip top-right when `hostChanged === true`; absent otherwise.

**Zone 2 — Destination.** 20px semibold hostname. Below: full URL in 12px muted, `text-overflow: ellipsis`, hover tooltip shows full string. Click opens in new tab. Adjacent copy button writes the URL to the clipboard with a 1-second "Copied" toast.

**Zone 3 — Hop trail.** Collapsed by default. Single muted line: `↳ N hops · show trail`. Expands to a vertical list of hops, each row `[METHOD] [STATUS] hostname/path` with status code color-coded. For 1-hop resolves the trail collapses entirely. LinkedIn interstitial extractions add an "extracted from interstitial" footnote on the final hop.

**States.** Idle: card hidden, status line shows placeholder. Loading: skeleton card with shimmer (CSS `@keyframes`). Error: red top strip, error message in zone 2, no zones 1/3.

## Domain Health card (risk-first)

Triggered by full-width borderless "Check Domain Health" button below the hero card. Loading: pulsing skeleton with text *"Checking certificates and registration…"*.

When loaded:

- Filled pill badge `LOW` / `MEDIUM` / `HIGH` in `--risk-*`.
- Plain-language reason sentence next to the badge, generated by new `summarizeRisk(health)` helper. Picks the dominant signal: recent registration > low cert count > established baseline.
- Muted link "Show certificate & registration details" expands a two-column key/value grid with existing crt.sh and RDAP fields, grouped under uppercase `CERTIFICATES` and `REGISTRATION` labels. Per-source errors render as red inline notes inside the relevant group, not as a top-level failure.

## History drawer

Slide-down from below the toolbar via `max-height` + `transform` (~200ms ease-out). Each entry is one row: neutral globe icon, hostname, relative timestamp (*"3m ago"*), small risk dot if a domain-health snapshot is stored. Click repopulates the result card from the stored entry. Empty: centered muted *"No resolved URLs yet."* No per-entry delete in this pass.

## Motion budget

Three animations only: history drawer slide, skeleton shimmer, copy toast fade. Theme transitions are instant. `prefers-reduced-motion: reduce` disables all three.

## Risk notes

- **Risk reasoning quality.** `summarizeRisk` is heuristic, not authoritative. The sentence is intentionally hedged (e.g. *"Registered 12 days ago"*) so users still see the underlying numbers if they expand the details grid.
